But something I usually try prior to complete re-install is a user profile refresh. Log in as the administrator (assuming you use some other account routinely), change the name of your routine user's folder in documents and settings (I like renaming over deleting), log out and log back in with your routine user. It will recreate your profile (filesystem + registry) and clear out many obnoxious and otherwise impossible to find erroneous/inconsistent states. ---

So much for trying to solve this by attempting to change my visual studio configuration. As soon as I tried to start the Visual Studio 2005 installation to add the missing components I was greeted by the infamous error:


d:\vs\vs_setup.msi could not be opened.

Uninstalling VS2005 to reinstall wouldn't work, either; it does use the same setup after all. So now I have the pleasure of having to start setting up my development virtual machine from scratch courtesy of a screwed VS installation. Thanks goodness this is just a virtual machine and not my main one.
